# 5. 多算法同步对比系统

## 5.1 功能目标

多个算法在同一地图同步运行。

---

## 5.2 对比模式

### 双窗口模式

```text
┌───────────────┬───────────────┐
│ BFS           │ A*            │
└───────────────┴───────────────┘
```

---

### 四窗口模式

```text
┌─────────┬─────────┐
│ BFS     │ DFS     │
├─────────┼─────────┤
│ Dijkstra│ A*      │
└─────────┴─────────┘
```

---

## 5.3 支持算法

| 算法       | 类型                |
| -------- | ----------------- |
| BFS      | 广度搜索              |
| DFS      | 深度搜索              |
| Dijkstra | 最短路径              |
| A*       | 启发式搜索             |
| JPS      | Jump Point Search |
| Theta*   | 平滑路径              |
| D* Lite  | 动态重规划             |

---

## 5.4 实时统计信息

| 指标         | 描述      |
| ---------- | ------- |
| 搜索节点数      | 总访问节点   |
| 路径长度       | 最终路径    |
| 算法耗时       | 执行时间    |
| OPEN 集大小   | 当前待搜索节点 |
| CLOSED 集大小 | 已搜索节点   |

# 14. 高性能渲染架构

## 14.1 Canvas 强制方案

禁止：

```text
一个格子 = 一个组件
```

原因：

* 大地图会导致组件爆炸
* UI 性能急剧下降

必须：

* Canvas 自绘
* 分层渲染

---

## 14.2 渲染层设计

```text
Canvas
 ├── BackgroundLayer
 ├── GridLayer
 ├── TerrainLayer
 ├── ObstacleLayer
 ├── HeatMapLayer
 ├── PathLayer
 └── AgentLayer
```

---

## 14.3 优化重点

| 优化项  | 目标          |
| ---- | ----------- |
| 局部刷新 | 减少重绘        |
| 批量绘制 | 减少 drawCall |
| 对象池  | 降低 GC       |
| 动画节流 | 稳定 FPS      |
| 数据缓存 | 提高效率        |

---
